## Overview

The Fee Engine Admin UI lets you create, test, and manage fee rules that determine how charges are calculated for payment transactions. Rules can be created manually via a form or generated by the AI assistant from a plain-English description.

Every rule is tested with a **Dry Run** before it goes live. Rules go live only when explicitly approved.

---

## Navigation

The top navigation bar has three sections:

| Link | Purpose |
|------|---------|
| **Fee Rules** | View and manage all live fee rules |
| **AI Drafts** | View and manage AI-generated rule drafts |
| **Dry Run** | Test a fee calculation without saving anything |

---

## Fee Rules

### Viewing and Filtering Rules

The **Fee Rules** page lists all rules in the system. Use the filter bar at the top to narrow results:

- **Payment Type** — e.g. DOMESTIC, INTERNATIONAL
- **Scheme** — FPS, BACS, CHAPS, SWIFT
- **Charge Bearer** — who pays the fee
- **Fee Type** — FLAT, PERCENTAGE, TIERED_SLAB, TIERED_STEP, FREE
- **Currency** — 3-letter ISO code (e.g. GBP, USD)
- **Destination Country** — 2-letter country code (international rules only)
- **Status** — Active / Inactive / All

Click **Clear** to reset all filters. The list pages through results using **Previous** and **Next**.

---

### Creating a Rule Manually

1. Click **New Rule** on the Fee Rules page.
2. Fill in the required fields (see [Rule Fields Reference](#rule-fields-reference)).
3. Optionally enter an **Instructed Amount** and **Currency** at the bottom to preview the calculated charge.
4. Click **Save** to create the rule immediately as an active live rule.

> **Note:** Rules created through the form go live immediately without a draft approval step. Use [AI Drafts](#ai-drafts) if you want a review-before-publish workflow.

---

### Editing a Rule

Click **Edit** on any rule in the list. The same form opens pre-filled with the existing values. Save to apply changes immediately.

---

### Activating and Deactivating Rules

Each rule in the list has an **Activate** or **Deactivate** button. Deactivated rules are ignored by the fee engine — no charges are calculated against them. Reactivate at any time.

---

## Dry Run

The **Dry Run** page lets you test any fee rule against a payment scenario without saving anything to the database.

**How to use it:**

1. Navigate to **Dry Run**.
2. Fill in the **Rule Definition** fields (same as the rule form).
3. Optionally provide:
   - **Instructed Amount** and **Currency** — to see calculated charges
   - **Debtor Account** and **Creditor Account** — to test account-specific matching
4. Click **Run**.

The result panel shows:
- A **green box** if the rule structure is valid and calculation succeeded
- A **red box** if validation or calculation failed, with an error description
- The calculated **charges** (charge type, amount, currency, who pays) when an instructed amount was provided

Leaving the instructed amount blank validates the rule structure only — no charges are calculated.

## AI Drafts

The AI assistant generates valid fee rule JSON from a plain-English description. All AI-generated rules go through a **draft workflow** before they become live rules.

### Draft Lifecycle

```
PENDING → DRY_RUN_FAILED → (edit) → PENDING
       ↘ DRY_RUN_PASSED → APPROVED (live rule created)
                        ↘ REJECTED
```

| Status | Meaning | Available Actions |
|--------|---------|-------------------|
| **PENDING** | Draft created, awaiting dry run | Dry Run, Edit, Reject, Delete |
| **DRY_RUN_FAILED** | Dry run found errors | Dry Run, Edit, Reject, Delete |
| **DRY_RUN_PASSED** | Dry run succeeded | Dry Run (retry), Approve, Reject, Delete |
| **APPROVED** | Pushed to fee engine as a live rule | — |
| **REJECTED** | Manually discarded | Delete |

---

### Generating a New Rule with AI

1. Click **New Draft** on the AI Drafts page.
2. Leave **Mode** as **GENERATE**.
3. Type your prompt (up to 2,000 characters) describing the rule you want.
4. Click **Generate**.

The AI creates a draft and redirects you to the **Draft Detail** page. See [Sample AI Prompts](#sample-ai-prompts) for ready-to-use examples.

---

### Updating an Existing Rule with AI

1. Click **New Draft** on the AI Drafts page.
2. Change **Mode** to **UPDATE**.
3. Use the **Target Rule** picker to select the rule you want to modify.
4. Type your prompt describing what to change.
5. Click **Generate**.

The AI receives the current rule JSON plus your instructions and returns an updated version.

**Example prompt:**
> Increase the flat amount to 2.50 GBP and change the scheme to CHAPS.

---

### Draft Detail and Lifecycle

After generation, the **Draft Detail** page shows:

- **Prompt** — what you asked the AI
- **AI Explanation** — the AI's plain-English summary of the rule
- **Rule JSON** — the generated rule (click **Edit Rule** to modify via form)
- **AI Review** — click this button to ask the AI to analyse the rule for potential issues
- **Payment Context** — enter an instructed amount and currency to test a specific scenario
- **Dry-run Result** — appears after running; green = passed, red = failed

**Workflow:**

1. Read the AI explanation and review the Rule JSON.
2. Click **AI Review** if you want a second opinion from the AI.
3. If the rule needs changes, click **Edit Rule** to adjust via the form. Editing resets the draft to PENDING.
4. Enter an **Instructed Amount** (e.g. `5000`) and **Currency** (e.g. `USD`) in the Payment Context section.
5. Click **Dry Run** to validate and calculate charges.
6. Change the amount and click **Dry Run** again to test different scenarios — you can retry as many times as you like.
7. When satisfied, click **Approve** to push the rule live.

> **Tip:** You must run a dry run and get a PASSED result before the Approve button appears.

---

### Approving a Draft

Click **Approve** when the draft status is **DRY_RUN_PASSED**. This:

1. Creates a new active fee rule in the engine.
2. Links the draft to that rule (the rule ID is shown on the draft detail page).
3. Sets the draft status to **APPROVED**.

Approved drafts cannot be deleted or edited.

## Rule Fields Reference

| Field | Required | Description |
|-------|----------|-------------|
| **Payment Type** | Yes | The type of payment this rule applies to (see below) |
| **Scheme** | Yes | The payment scheme: FPS, BACS, CHAPS, or SWIFT |
| **Charge Bearer** | Yes | Who pays the fee: BorneByDebtor, BorneByCreditor, Shared, FollowingServiceLevel |
| **Charge Type** | Yes | A label for the charge (e.g. `ServiceCharge`, `WireFee`). Used to group charges in the output |
| **Fee Type** | Yes | How the fee is calculated (see below) |
| **Currency** | Yes | 3-letter ISO currency code (e.g. GBP, USD, EUR) |
| **Priority** | No | Integer ≥ 0. When multiple rules match, the highest priority wins. Default: 0 |
| **Account Identification** | No | Restrict the rule to a specific account number |
| **Destination Country** | No | 2-letter country code (e.g. GB, US). Only valid for INTERNATIONAL, INTERNATIONAL_SCHEDULED, INTERNATIONAL_STANDING_ORDER |

### Payment Types

| Value | Description |
|-------|-------------|
| DOMESTIC | Standard domestic payment |
| FILE | Bulk file payment |
| INTERNATIONAL | International wire transfer |
| DOMESTIC_SCHEDULED | Future-dated domestic payment |
| DOMESTIC_STANDING_ORDER | Recurring domestic payment |
| INTERNATIONAL_SCHEDULED | Future-dated international payment |
| INTERNATIONAL_STANDING_ORDER | Recurring international payment |

### Fee Types

| Type | Description | Required Fields |
|------|-------------|-----------------|
| **FLAT** | Fixed charge regardless of amount | `flatAmount` |
| **PERCENTAGE** | Charge as a percentage of the transaction amount | `percentage` (decimal, e.g. `0.005` for 0.5%) |
| **TIERED_SLAB** | The entire amount falls into one bracket; that bracket's rate applies to the full amount | `tiers` |
| **TIERED_STEP** | Progressive: every bracket the amount passes through fires; each applies to its portion only; results are summed | `tiers` |
| **FREE** | No charge | _(none)_ |

**PERCENTAGE only:** optionally set `Min Fee` and `Max Fee` to cap the calculated charge.  
**TIERED_SLAB / TIERED_STEP:** `Min Fee`, `Max Fee`, `flatAmount`, and `percentage` must be left blank.

## Tier Configuration

Tiers are used with **TIERED_SLAB** and **TIERED_STEP** fee types.

### How Brackets Work

Each tier defines a bracket `[Min, Max)` and a formula to apply within that bracket.

| Fee Type | Which tiers fire | Formula applies to |
|----------|-----------------|-------------------|
| TIERED_SLAB | The **one** bracket where `Min < amount ≤ Max` | The **full** transaction amount |
| TIERED_STEP | **Every** bracket where `amount > Min` | Only the **portion** of the amount within that bracket |

**TIERED_STEP example** — 3 tiers: 3% (0–10,000), 2% (10,000–50,000), 1% (50,000+):

| Payment amount | Tier 1 contribution | Tier 2 contribution | Tier 3 contribution | Total fee |
|---------------|--------------------|--------------------|--------------------|----|
| 5,000 | 5,000 × 3% = 150 | — | — | **150** |
| 10,000 | 10,000 × 3% = 300 | — | — | **300** |
| 30,000 | 10,000 × 3% = 300 | 20,000 × 2% = 400 | — | **700** |
| 60,000 | 10,000 × 3% = 300 | 40,000 × 2% = 800 | 10,000 × 1% = 100 | **1,200** |

### Boundary Rule

A tier fires only when `amount > tier Min` (strictly greater than). An amount **equal** to a boundary belongs to the **lower** bracket.

> **Example:** With tiers [0 – 1,000] and [1,000 – 10,000], an amount of exactly **1,000** is charged only by the first tier.

### Rate Types per Tier

| Rate Type | Formula | Required Fields |
|-----------|---------|----------------|
| **FIXED** | Flat amount per bracket entered | `amount` |
| **PERCENTAGE** | `bracket × rate` | `percentage` (e.g. `0.03` for 3%) |
| **HYBRID** | `amount + (bracket × rate)` | `amount` + `percentage` |
| **GREATER_OF** | `max(amount, bracket × rate)` | `amount` + `percentage` |

### Tier Setup Tips

- Use **whole numbers** for Min and Max boundaries (e.g. `1000`, not `999.99`).
- Adjacent tiers must share the same boundary value — set the previous tier's Max equal to the next tier's Min (e.g. `0 / 1000`, `1000 / 10000`).
- Set the **last tier's Max to `999999999`** to catch all amounts above the final boundary.
- You can mix rate types across tiers (e.g. FIXED for low amounts, PERCENTAGE for higher amounts).

## Sample AI Prompts

Copy any prompt below into the **AI Drafts → New Draft** form. Adjust names, amounts, and currencies as needed.

---

### FLAT Fee

**Domestic FPS — fixed handling fee**
> Create a FLAT fee rule called `DOMESTIC_HANDLING` for domestic FPS payments, charged to the debtor, flat amount 0.25 GBP, priority 0.

**CHAPS high-value — flat service charge**
> Create a FLAT fee rule called `CHAPS_SERVICE` for domestic CHAPS payments, charged to the debtor, flat amount 15.00 GBP, priority 5.

**International wire — fixed processing fee**
> Create a FLAT fee rule called `INTL_PROCESSING` for international SWIFT payments, charged to the debtor, flat amount 8.00 USD.

---

### PERCENTAGE Fee

**Basic percentage with no caps**
> Create a PERCENTAGE fee rule called `FX_MARKUP` for international SWIFT payments, charged to the debtor, rate 1.5% (percentage 0.015), currency USD.

**Percentage with min and max caps**
> Create a PERCENTAGE fee rule called `FPS_PCT_FEE` for domestic FPS payments, charged to the debtor, rate 0.5% (percentage 0.005), minimum fee 0.10 GBP, maximum fee 5.00 GBP, currency GBP.

**BACS percentage with floor**
> Create a PERCENTAGE fee rule called `BACS_FEE` for domestic BACS payments, charged to the debtor, rate 0.3% (percentage 0.003), minimum fee 0.05 GBP, maximum fee 10.00 GBP, currency GBP.

---

### TIERED_SLAB Fee

> In TIERED_SLAB, the **entire** payment amount is charged at the rate of whichever single bracket it falls into.

**Flat amount per band (CHAPS)**
> Create a TIERED_SLAB fee rule called `CHAPS_BAND_FEE` for domestic CHAPS payments, charged to the debtor, in GBP, with 3 tiers:
> - Tier 1 min 0 max 1000: FIXED amount 1.00
> - Tier 2 min 1000 max 10000: FIXED amount 3.00
> - Tier 3 min 10000 max 999999999: FIXED amount 10.00

**Percentage per band (international)**
> Create a TIERED_SLAB fee rule called `SWIFT_SLAB_PCT` for international SWIFT payments, charged to the debtor, in USD, with 3 tiers:
> - Tier 1 min 0 max 5000: PERCENTAGE percentage 0.02
> - Tier 2 min 5000 max 50000: PERCENTAGE percentage 0.015
> - Tier 3 min 50000 max 999999999: PERCENTAGE percentage 0.01

**Hybrid slab (flat + percentage)**
> Create a TIERED_SLAB fee rule called `BACS_HYBRID_SLAB` for domestic BACS payments, charged to the debtor, in GBP, with 2 tiers:
> - Tier 1 min 0 max 1000: HYBRID amount 1.00 percentage 0.005
> - Tier 2 min 1000 max 999999999: HYBRID amount 3.00 percentage 0.003

---

### TIERED_STEP Fee

> In TIERED_STEP, every bracket the payment passes through contributes to the total — like a progressive income tax.

**Pure percentage (progressive)**
> Create a TIERED_STEP fee rule called `PROGRESSIVE_SWIFT` for international SWIFT payments, charged to the debtor, in GBP, with 3 tiers:
> - Tier 1 min 0 max 10000: PERCENTAGE percentage 0.03
> - Tier 2 min 10000 max 50000: PERCENTAGE percentage 0.02
> - Tier 3 min 50000 max 999999999: PERCENTAGE percentage 0.01

**Mixed rate types (entry fee + progressive)**
> Create a TIERED_STEP fee rule called `WIRE_STEP_FEE` for international SWIFT payments, charged to the debtor, in USD, with 4 tiers:
> - Tier 1 min 0 max 1000: FIXED amount 5.00
> - Tier 2 min 1000 max 10000: PERCENTAGE percentage 0.005
> - Tier 3 min 10000 max 100000: HYBRID amount 10.00 percentage 0.003
> - Tier 4 min 100000 max 999999999: GREATER_OF amount 50.00 percentage 0.002
>
> Do not include minFee or maxFee.

**Standing orders — step fee**
> Create a TIERED_STEP fee rule called `DOM_SO_FEE` for domestic standing order payments (DOMESTIC_STANDING_ORDER), scheme FPS, charged to the debtor, in GBP, with 2 tiers:
> - Tier 1 min 0 max 500: FIXED amount 0.10
> - Tier 2 min 500 max 999999999: PERCENTAGE percentage 0.001
>
> Do not include minFee or maxFee.

---

### FREE Fee

**Free domestic FPS**
> Create a FREE fee rule called `FPS_FREE` for domestic FPS payments, charged to the debtor, currency GBP.

**Free international EUR transfers**
> Create a FREE fee rule called `EUR_WAIVER` for international SWIFT payments, charged to the debtor, currency EUR, destination country DE.
